The rising relevance of Cloud Native Programming

Friday, Jun 12, 2020| Tags: data, analytics, tech, cloud

I got to learn about Cloud Native Computing Foundation as I was browsing through Helm. From CNCF website:

The Cloud Native Computing Foundation (CNCF) hosts critical components of the global technology infrastructure. CNCF brings together the worldâ€™s top developers, end users, and vendors and runs the largest open source developer conferences. CNCF is part of the nonprofit Linux Foundation.

It is also important to understand the meaning of cloud native computing and what the approach covers / recommends. From wiki:

Cloud native computing is an approach in software development that utilizes cloud computing to “build and run scalable applications in modern, dynamic environments such as public, private, and hybrid clouds” [1]. Technologies such as containers, microservices, serverless functions and immutable infrastructure, deployed via declarative code are common elements of this architectural style.

Keys:

  1. a software development approach
  2. utilizing cloud services that are modern and dynamic (public, private and hybrid)
  3. build and run scalable applications
  • microservices wrapped in containers, serverless functions, declarative code deployment

The recent data lake architecture aims to be hybrid, one in which on-premises and cloud infra-structure exist using a combination of OSS technology. Kubernetes for container orchestration, Spark for data processing, Azure Datalake or S3 for storage is gaining mainstream momentum.
